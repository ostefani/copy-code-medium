Data collection & EU users / GDPR

This extension does not collect, transmit, or store any personal data or other user data. All operations: detecting code blocks and copying text are performed locally in the browser on the pages you visit. The extension does not use third-party analytics, telemetry, or advertising services.

Because no personal data is collected or processed by the extension in normal operation, the developer is not acting as a data controller for extension usage. The source code is public and available on GitHub.

Permissions

This extension does not request any optional extension permissions in the manifest. It injects a content script into Medium article pages so the extension can detect code blocks and insert copy buttons. As a result, Chrome will indicate that the extension can "Read and change site data" for medium.com and subdomains.
These capabilities are used solely to interact with the page DOM and render the copy UI. No page content is collected, transmitted, or stored.

Third parties

This extension does not use third-party analytics, tracking, advertising, or any other external services that receive user data. It does not share user data with any third parties.

Storage and retention

The extension stores no user data locally or remotely. No information is uploaded to servers or persisted outside the user's browser extension storage (and even then, only if explicitly implemented and described).

Security

Because extension does not transmit or store user data, there is no personal data at risk. The code follows common best practices: no hard-coded secrets, no unexpected network uploads, and no use of untrusted external scripts.

Changes to this policy

If the extension's functionality, permissions, or data-handling practices change in any way that affects user privacy, this policy will be revised to reflect those changes. The revised policy will be published at this same URL with an updated "Last updated" date; for questions about changes, contact mailto:me@ostefani.aleeas.com.
